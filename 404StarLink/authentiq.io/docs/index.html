Authentication

Up 
get /authorize
Authenticate a user (authorize)

Start a session with Authentiq Connect to authenticate a user.

GET https://connect.authentiq.io/authorize?client_id=<your-client-id>&response_type=code+id_token&scope=openid+email&redirect_uri=<your-redirect-uri>&state=0123456789

This endpoint also supports the POST method.

Query parameters

client_id (required)
Query Parameter — A client ID obtained from the Dashboard. default: null
response_type (required)
Query Parameter — The OIDC response type to use for this authentication flow. Valid choices are code, id_token, token, token id_token, code id_token code token and code token id_token, but a client can be configured with a more restricted set. default: null
scope (required)
Query Parameter — The space-separated identity claims to request from the end-user. Always include openid as a scope for compatibility with OIDC. default: null
redirect_uri (required)
Query Parameter — The location to redirect to after (un)successful authentication. See OIDC for the parameters passed in the query string (response_mode=query) or as fragments (response_mode=fragment). Unless the client is in test-mode this must be one of the registered redirect URLs. default: null
state (required)
Query Parameter — An opaque string that will be passed back to the redirect URL and therefore can be used to communicate client side state and prevent CSRF attacks. default: null
response_mode (optional)
Query Parameter — Whether to append parameters to the redirect URL in the query string (query) or as fragments (fragment). This option usually has a sensible default for each of the response types. default: null
nonce (optional)
Query Parameter — An nonce provided by the client (and opaque to Authentiq Connect) that will be included in any ID Token generated for this session. Clients should use the nonce to mitigate replay attacks. default: null
display (optional)
Query Parameter — The authentication display mode, which can be one of page, popup or modal. Defaults to page. default: page
prompt (optional)
Query Parameter — Space-delimited, case sensitive list of ASCII string values that specifies whether the Authorization Server prompts the End-User for reauthentication and consent. The supported values are: none, login, consent. If consent the end-user is asked to (re)confirm what claims they share. Use none to check for an active session. default: login
max_age (optional)
Query Parameter — Specifies the allowable elapsed time in seconds since the last time the end-user was actively authenticated. default: 0
ui_locales (optional)
Query Parameter — Specifies the preferred language to use on the authorization page, as a space-separated list of BCP47 language tags. Ignored at the moment. default: null

Responses

302

A successful or erroneous authentication response.

303

Sign in with Authentiq page, popup or modal.
Up 
post /token
Obtain an ID Token (token)

Exchange en authorization code for an ID Token or Access Token.

This endpoint supports both client_secret_basic (default) and client_secret_basic authentication methods, as specified by the client's token_endpoint_auth_method.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/x-www-form-urlencoded

Request headers

Authorization (optional)
Header Parameter — HTTP Basic authorization header. default: null

Form parameters

client_id (required)
Form Parameter — The registered client ID. default: null
client_secret (required)
Form Parameter — The registered client ID secret. default: null format: password
code (required)
Form Parameter — The authorization code previously obtained from the Authentication endpoint. default: null
grant_type (required)
Form Parameter — The authorization grant type, must be authorization_code. default: null
redirect_uri (required)
Form Parameter — The redirect URL that was used previously with the Authentication endpoint. default: null

Return type

Token

get /{client_id}/iframe
Include a session iframe (authorizeIframe)

An OpenID Connect Session Management iframe to facilitate e.g. single sign-on or remote logouts.

The iframe implements the OIDC postMessage-based change notification protocol via which a client can receive notifications about session state changes.

Path parameters

Path Parameter — Client identifier default: null
