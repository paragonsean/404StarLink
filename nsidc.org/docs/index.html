get /Facets
View the facet information corresponding to a search (facets)
In the NSIDC Search and Arctic Data Explorer interfaces, this endpoint is used in conjunction with /OpenSearch whenever a user submits a new search. Consequently, it has the same parameters as /OpenSearch.

Query parameters

searchTerms (optional)
Query Parameter — URL-encoded keyword or keywords desired by the client; OpenSearch 1.1 default: null
count (optional)
Query Parameter — The number of search results per page desired by the client; OpenSearch 1.1 default: 25
startIndex (optional)
Query Parameter — First search result desired by the search client; OpenSearch 1.1 default: 1
spatial (optional)
Query Parameter — 4 comma separated values - W, S, E, N; OpenSearch-Geo 1.0, "box" parameter default: -180.0,-90.0,180.0,90.0
sortKeys (optional)
Query Parameter — Sort the results by most relevant (default), smallest or largest spatial area, shortest or longest temporal duration, or most recently updated; partial implementation of OpenSearch SRU 1.0 default: score,,desc
startDate (optional)
Query Parameter — The start date in yyyy-mm-dd format default: null format: date
endDate (optional)
Query Parameter — The end date in yyyy-mm-dd format default: null format: date
facetFilters (optional)
Query Parameter — Describes faceted restrictions on the search. A URL-encoded JSON object where the keys are the names of the facet, and the values are arrays of the selected facet values default: null
source (optional)
Query Parameter — Custom parameter for selecting which source to use; the Arctic Data Explorer (ADE) uses data aggregated from many sources, including, but not limited to, NSIDC default: NSIDC

Return type

String

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/nsidcfacets+xml

Responses

get /suggest
Suggest search terms based on a partial query (id)
In NSIDC Search and the Arctic Data Explorer, this endpoint is queried whenever the user types into the search terms box, and the returned suggestions are displayed in a dropdown beneath the search terms box. The q parameter and returned JSON follow the specifications of the OpenSearch Suggestions 1.0 extension.

Query parameters

q (required)
Query Parameter — Search terms typed into the interface (minimum two characters) default: null
source (required)
Query Parameter — Custom parameter for selecting which source to use; the Arctic Data Explorer (ADE) uses data aggregated from many sources, including, but not limited to, NSIDC default: NSIDC

Return type

String

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/x-suggestions+json

Responses
